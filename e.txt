/*
既然要贴满，比如长宽都要贴满，那么我们求出来长、宽上分别需要几个，然后乘一块就是答案了
看长，设长为n。
n=5 k=3  则需要2块
n=6 k=3  则需要2块
n=7 k=3  则需要3块
所以结果要向上取整，算宽的时候也是一样。
教给大家一个向上取整的技巧。
长为n，广告是k，则在长上需要：(n+k-1)/k块。
大家可以验证一下。
*/



广告牌的形状是个n*m的矩形，而广告牌上的广告可以看成k*k的广告（即每种广告都是k*k的），当然贴的时候广告是不重叠的，都是边挨着边的，并且将整个广告牌的版面都全部被覆盖（当然广告可以不整体地贴在牌上，因为k可能不能被n或者m整除），问需要多少广告能把广告牌贴满且不能重叠。

Input
输入的第一行是一个整数T（T<=30），表示测试数据的组数：
每组数据包含三个整数n，m，k，分别代表广告牌的长，宽和广告的边长。(1<=n,m,k<=1000)

Output
每组数据输出一个整数表示答案，即需要的广告数。




#include <stdio.h>
#include <stdio.h>
int main()
{
    int n,m,k,T;
    scanf("%d",&T);
    while(T--){
        scanf("%d%d%d",&n,&m,&k);
        int x=(n+k-1)/k;	//矩形的长上需要x块
        int y=(m+k-1)/k;	//矩形的宽上需要y块
        printf("%d\n",x*y);
    }
    return 0;
}
